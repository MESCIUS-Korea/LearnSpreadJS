필터 동작 | Filter Action

Filter 개체의 onFilter 메서드는 사용자 지정 필터 동작을 만드는 기능을 제공합니다.

1. RowFilterBase에서 확장 된 새 클래스를 만듭니다. 예를들면 : 

function SelectionFilter (range) {
GC.Spread.Sheets.Filter.RowFilterBase.call(this, range);
}
SelectionFilter.prototype = new GC.Spread.Sheets.Filter.RowFilterBase();

2. 다음 단계는 onFilter(args)를 대체하여 사용자 정의 필터를 적용하는 것입니다. args.action은 다음 FilterActionType중 하나의 요소를 가집니다.

  • filter
  • unfilter
SelectionFilter.prototype.onFilter = function (args) {
   var sheet = args.sheet;
   if (args.action === GC.Spread.Sheets.Filter.FilterActionType.filter) {
       sheet.setValue(9, 1, "필터링 된 아이템을 선택 하세요");
   }else {
       sheet.setValue(9, 1, "필터링된 아이템이 없습니다.");
   }
};

이 기능을 사용하여 현재 필터가 적용되어 있는지 확인하고 추가로 필터를 적용할지 또는 필터를 제거해야하는지 결정하시면 됩니다. 예를들면:

filterButtonVisble 메서드를 사용하여 필터 버튼을 표시할지 여부를 결정할 수 있습니다.
